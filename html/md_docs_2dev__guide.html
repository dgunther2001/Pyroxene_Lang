Developer Guide

Note that additional developer specific instructions are located below.

Clone the repository

  • git clone https://github.com/dgunther2001/pyroxene_lang

Install and enable Docker and/or Docker Desktop

  • GUI: Docker
  • CLI:
      - On Ubuntu CLI:
        - sudo apt-get install -y apt-transport-https ca-certificates curl software-properties-common
        - curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
        - echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
        - sudo apt-get update
        - sudo apt-get install -y docker-ce docker-ce-cli containerd.io
        - sudo systemctl start docker
        - sudo systemctl enable docker

Execution

Zsh and Bash Shells

  • Running Language Without Aliases:
      - Debug Mode:
        - chmod u+x run_debug_driver.sh
        - ./run_debug_driver.sh /path/to/your/pyroxene/file
      - Regular Mode:
        - chmod u+x run_driver.sh
        - ./run_driver.sh /path/to/your/pyroxene/file
  • Running Language With Aliases:
      - chmod u+x run_debug_driver.sh run_driver.sh setup.sh
      - ./setup.sh
      - Bash: source ~/.bashrc
      - Zsh: source ~/.zshrc
      - Debug Mode:
        pyroxene /path/to/your/pyroxene/file
      - Regular Mode:
        pyroxene-debug /path/to/your/pyroxene/file

Debug Mode

  • Running Through LLDB:
      - chmod u+x run_lldb.sh
      - Add alias to shell config file: ‘alias pyroxene-lldb=’/Users/danielgunther/Developer/pyroxene_lang/run_lldb.sh'\ilinebr<br> &emsp; - **Bash:**source ~/.bashrc\ilinebr<br> &emsp; - **Zsh:**source ~/.zshrc\ilinebr<br> &emsp; -pyroxene-lldb /path/to/your/pyroxene/file`
  • Various Debug Modes in Debug Driver:
      Note that these are modified via a macro at the top of /debug_test_suite/unit_test_driver.cpp called DEBUG_OPTION
      1: Token Stream.
      2: Various associated values storage array to validate that each row contains at most 1.
      3: Print tokens and run primary driver loop.
      4. Debug associated value lexer array.
      5: Print LLVM Module and run it through the JIT.
      6: Print the LLVM Module and don't execute it.
      7: Performance output.
      Note that a parse tree can be printed by turnin the PARSER_PRINT_UTIL macro un /include/utility.h to 1.
  • Running Through Valgrind:
      Note that on segfault, the Valgrind Address Sanitizer will automatically be run.
      To force it to run and view the console output, go to the primary Dockerfile.
      Uncomment the bottom two lines indicating Valgrind, and comment out the current CMD abnd ENTRYPOINT lines.
